# Reward Repo Ops

This directory is the in-repository staging location for reward and governance automation that does not belong to the product delivery baseline.

Current status:

- workflows remain in `.github/workflows/` but are manual-only
- scripts and templates live here instead of under `.github/`
- long-term target is a dedicated repo-ops or governance repository

Files in this directory are intentionally separated from application build, deploy, and fork-maintenance automation.
