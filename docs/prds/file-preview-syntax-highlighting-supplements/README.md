## 文档使用指南

### 开发阶段
1. **开始开发前**：阅读 PRD 和约束规范，理解功能边界和业务规则
2. **编写代码时**：参考影响分析，了解需要修改的文件
3. **编写测试时**：参考验收用例和测试计划，确保覆盖所有场景
4. **代码审查时**：检查是否满足所有业务规则（BR-XXX）

### 测试阶段
1. **单元测试**：按照测试计划执行，确保覆盖率 > 80%
2. **集成测试**：验证所有验收用例（AC-XXX）
3. **性能测试**：确保满足性能约束（500KB < 500ms）
4. **安全测试**：验证 XSS 防护（AC-S-XXX）

### 部署阶段
1. **部署前**：检查回滚计划是否就绪
2. **部署后**：监控错误率和性能指标
3. **问题排查**：参考风险评估（R-XXX）定位问题
